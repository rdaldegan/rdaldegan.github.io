| Conecta Brasal

2024 — Present

Context

Problem: Brasal, the Coca-Cola distributor in the Federal District and surrounding region of Brazil, needed to centralize all B2B customer relationships in a single platform — from product orders to debt payments and equipment requests.

Users: B2B customers (bars, restaurants, supermarkets), sales team and Brasal's internal administration.

My role

  • Backend: REST APIs, business rules, integration with internal systems
  • Frontend: mobile app and web panel interfaces
  • Infrastructure: container setup and orchestration with Docker Swarm

Technologies

Laravel 10 React Native React.js MySQL Docker Swarm

Architecture

System split into 3 independent microservices, each with its own Laravel backend:

  • Users: authentication, registration and customer profile management
  • Sales: product catalog, cart, orders and debt payments
  • Campaigns: admin panel, promotions and commercial campaign management

Deployed with Docker Swarm for container orchestration, with replicas and load balancing to ensure availability and scalability.

Technical challenges

  • Ensuring high availability for a system that processes all Coca-Cola sales in the region
  • Orchestrating 3 independent modules with consistent communication between them
  • Scaling horizontally with Docker Swarm and load balancing across container replicas

Solution

Microservices architecture with clear separation of responsibilities between Users, Sales and Campaigns modules. Each service runs in Docker containers managed by Swarm, which handles automatic replication and load distribution. This allowed each module to scale independently based on demand.

Results

  • 12,000+ customers registered on the platform
  • R$ 50M+ average weekly revenue
  • Single platform for the entire B2B commercial flow in the Federal District
  • Available as mobile app (iOS/Android) and web version

| Customer Portal — Brasal Incorporações

2025 — Present

Context

Problem: Real estate buyers from Brasal Incorporações needed a unified channel to track their properties, pay installments, schedule visits and open technical support tickets — all digitally.

Users: Real estate buyers from Brasal Incorporações.

My role

  • Complete mobile app in React Native (64 screens)
  • Web dashboard in Vue.js 3
  • REST API in Laravel Lumen with legacy SOAP system integration
  • Integration with legacy Oracle database and MySQL

Technologies

React Native Expo Vue.js 3 Laravel Lumen MySQL Oracle DB AWS S3 Salesforce OneSignal Docker

Architecture

Distributed system in 3 independent layers: mobile app (React Native/Expo), web dashboard (Vue.js 3) and backend API (Laravel Lumen). The API serves as a bridge between modern frontends and legacy company systems via SOAP (32+ WSDL operations).

  • Dual Database: MySQL for app data + Oracle for legacy contract and financial data
  • Integrations: Salesforce (CRM), OneSignal (push), AWS S3 (documents), SendGrid (emails)
  • Authentication: JWT with biometric support (Face ID / fingerprint) on mobile

Technical challenges

  • Integrating with legacy SOAP web services (20+ years old) without compromising user experience
  • Keeping two databases (MySQL + Oracle) synchronized and consistent
  • Complex financial calculations: late interest, installment prepayment, discount simulations
  • Cross-platform biometric authentication (Android and iOS)
  • Push notifications at scale for thousands of buyers

Solution

I built a SOAP abstraction layer in the backend that translates REST calls into WSDL operations, with caching to reduce latency. The Repository + Service pattern isolates access to both databases, enabling gradual migration from Oracle to MySQL. On mobile, a custom design system with 40+ components ensures visual consistency, and Recoil manages state atomically. Job queues process notifications and emails asynchronously.

Results

  • App available on iOS and Android + web dashboard
  • Complete flow: from property lookup to installment payments
  • Salesforce integration for scheduling and technical support
